# AmpDock Implementation Plan

**Goal:** Deliver a packaged Windows desktop player with three detachable, dockable windows and a freely resizable playlist.

**Architecture:** Electron main process owns windows, file access, metadata, persistent state, and docking. A sandboxed preload exposes narrow APIs. One renderer supports `player`, `equalizer`, and `playlist` modes. The player renderer owns audio and Web Audio nodes; state is synchronized through the main process.

**Tech stack:** Electron 43, vanilla HTML/CSS/JavaScript, Web Audio API, music-metadata, Lucide, Node test runner, electron-builder.
